Auctions

To benefit from the use of |es|f| you should be familiar with the esniper terms "Auction" and "Bidding Group" and have used esniper successfully at least once from the console.

|es|f| is only a frontend that remotely controls and manages esniper!

Operation

|es|f| auctions are organized in three levels (one more than esniper itself).

For the following example, we have three auctions: a Graphics Card "ABC", a Graphics Card "CDE", and a CD "SomeSampler"

1. Category (Only in es-f)

The use of the Category level is optional. It is is a simple method of classifying related auctions into groups, eg "Computer", "Music" etc.

However, auctions are sorted for display by category. If the category level is not used, the auctions will be listed by date in a confused sort!

To assign a category, mark the checkbox on the left of the auction description. Then select "Move to category" in the drop down menu called "-Action-" at the bottom of the page.

(So we assign the two graphics cards the category "Hardware" and the CD the category "Music".)

Hardware
  Graphics Card ABC
  Graphics Card CDE
Music
  CD "SomeSampler"

2. Bidding Group (Multiple Auctions) (es-f and esniper)

Bidding groups are exactly the same as in esniper. Several similar auctions can be bid on as a group until the "requested" number of items have been won, or the amount available is exhausted.

The bid specified for the group applies for all auctions in the group that do not have individual bids set (E.g. An individual bid could be in a currency other than the default currency).

The auctions are now listed within the category by groups.

(So we assign the two graphics cards to the group "Graphics").

  Graphics Card ABC  \
                      >  Graphics
  Graphics Card CDE  /

3. Auction (es-f and esniper)

Each individual auction is the lowest level.

For simplicity, each auction is also a bidding group.

The auctions will be displayed in the group after the end date unless deleted.

In this example the CD is such a group with only one auction.

In the end, we have the following structure:

Hardware
  Graphics Card ABC  \
                      >  Graphics
  Graphics Card CDE  /
Music
  CD "SomeSampler"

The green start button indicates that this bidding group is ready to start. At this point, NO esniper process is running for this bid. It can be started if a group bid has been set.

Clicking the button will start esniper for this group which can also be seen in the "Processes" page and in the "Logs" page if all goes well.

If esniper detects an error it is recorded in the corresponding group log.

When |es|f| successfully starts esniper, the red stop button indicates that a process is underway and the Stop button can be clicked if it is necessary to stop it.
